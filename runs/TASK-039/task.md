## Description
Add a stable read-only API for the Quest Invite tab:

`GET /api/v1/missions/invite/overview?user_id=...`

The response must be stable enough for Mobile/Frontend to render the Invite tab without hardcoding. If there is no persisted config yet, return predictable defaults with `coming_soon=true` and null/zero counters rather than omitting fields.

## Acceptance Criteria
- [ ] `shared-lib/proto/missionspb/missions.proto` has an additive Invite overview RPC with grpc-gateway annotation `GET /api/v1/missions/invite/overview`.
- [ ] Generated `missionspb` artifacts are regenerated.
- [ ] `Games-Labs-Missions` exposes HTTP and gRPC-compatible handling for the Invite overview endpoint.
- [ ] Response includes stable fields: `enabled`, `coming_soon`, `title`, `description`, `rules`, `rewards_preview`, and `counters`.
- [ ] Missing config/data returns predictable defaults instead of shape drift.
- [ ] No referral flow, invite claim, or attribution behavior is implemented.
- [ ] Focused service/handler tests cover default response stability and fallback behavior.
- [ ] `docs/Games-Labs-APIs.postman_collection.json` includes the new request.

## Suggested Response Shape
```json
{
  "enabled": true,
  "coming_soon": true,
  "title": "Invite friends",
  "description": "Invite friends and unlock rewards.",
  "rules": [
    "Invite a friend",
    "Your friend joins",
    "Rewards become available when referral rewards launch"
  ],
  "rewards_preview": [
    { "type": "coin", "amount": 3000, "label": "+3,000" }
  ],
  "counters": {
    "invited": 0,
    "joined": 0,
    "required": 3
  }
}
```
