## Description
Add `GET /api/v1/vip/tiers` as a User-owned API. It should expose the VIP/Game level tier catalog derived from existing `level_configs` data plus code-level presentation presets for labels/groups/colors/perks. This keeps VIP/Game level ownership in `Games-Labs-User` and avoids route conflicts with Missions.

## Suggested Contract
Add to `userpb.proto`:

```proto
rpc ListVipTiers(google.protobuf.Empty) returns (ListVipTiersResponse) {
  option (google.api.http) = { get: "/api/v1/vip/tiers" };
}

message VipTier {
  int64 level = 1;
  string label = 2;
  string group = 3;
  string color = 4;
  int64 turnover_required = 5;
  int32 games_available = 6;
  repeated string perks = 7;
}

message ListVipTiersResponse {
  basepb.StatusResponse status = 1;
  repeated VipTier tiers = 2;
}
```

HTTP response should be compatible with:

```json
{
  "tiers": [
    {
      "level": 1,
      "label": "VIP1",
      "group": "Starter",
      "color": "Silver",
      "turnover_required": 0,
      "games_available": 5,
      "perks": ["Daily login bonus"]
    }
  ]
}
```
